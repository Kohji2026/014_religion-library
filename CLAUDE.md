## プロジェクト概要

### 目的
Excelに蓄積した宗教データ（28宗教×34指標）とGemini製インフォグラフィック（PNG・PDF・PPTX）を
一元的に閲覧できるパーソナル「宗教総合ライブラリ」を構築する。

### 設計思想
011_クラシック総合ライブラリと同じ「Excel→JSON→HTML」パイプライン。
3ビュー構成（宗教ビュー・歴史ビュー・比較ビュー）も同じ思想。

### データソース
- **Excelデータ**: `C:\Users\kohji\OneDrive\01_記録\22_宗教一覧.xlsx`（シート名:「宗教一覧」）
- **インフォグラフィック等**: `C:\Users\kohji\OneDrive\01_記録\頻度少\宗教\{宗教名}\{宗教名}.png / .pdf / .pptx`

### データ構造（Excel: シート「宗教一覧」）
- **基本情報**: 番号、宗教名、開祖、開宗年、分類、地域、信者数概算（億人）、説教者数、寺院・教会数、主な国
- **教義・実践**: 基本教義の数、特徴的な修行法、聖典、聖地、衣装、記号・象徴
- **社会的特性**: 戒律の厳しさ、世俗化度、国教化、政治勢力度、非宗教団体との相互作用、神聖性
- **組織・成長**: 派閥数、年間成長率(%)、コミュニティの結束度、社会との融和度
- **文化**: 学習難度、神話の複雑度、女性の地位、相続制度、出家制度、婚姻制度、食事タブー、備考

### 画面構成
1. **宗教ビュー**: 左サイドバーに宗教一覧、右に詳細（PNG＋情報グリッド）
2. **歴史ビュー**: 横スクロールタイムライン（開宗年バー表示）
3. **比較ビュー**: 全28宗教をソート可能テーブルで一覧

### データ更新フロー
1. `22_宗教一覧.xlsx` を編集（データ元帳）
2. `convert_to_json.py` を実行（`uv run --with openpyxl convert_to_json.py`）
3. `religion_data.json` が更新される
4. ブラウザで `宗教ライブラリ.html` をリロードするだけで反映

### ファイルパスについて
ローカルPNG・PDF・PPTXは `file:///` URLでアクセスする。
`file:///C:/Users/kohji/OneDrive/01_記録/頻度少/宗教/ユダヤ教/ユダヤ教.png` 形式。
Chromeでローカルファイルとして開いている場合は表示・リンク動作する。
