**作成日**: 2026-07-04（Cowork企画セッション）

**この文書が実装の正本。モックアップ_本棚.html は画面イメージ＋移植元コード。**
**レビュー済み**: UI/UXデザイナー・シニアエンジニアの2視点レビューを反映済み（v2）

## 1. 背景と目的

書籍検索.html には「検索起点」（全文検索・テーマ読み比べ）はあるが、
009_歴史検索のような **「本棚起点」＝本をそのまま読み始める入り口** がない。
検索語が空のときの空白エリアを本棚エントランスにする。

## 2. 確定仕様（grill-me 6論点・ユーザー承認済み）

| # | 論点 | 決定 |
|---|------|------|
| 1 | 閲覧方式 | 表紙クリック→**アプリ内ビューアがp1から開く**（原本PDFリンクはビューア内の既存ボタンで足りる） |
| 2 | 入り口の場所 | **書籍検索.htmlの🔍全文検索モード、検索語が空のときの結果エリア**。検索入力で結果一覧に切替、×クリアで本棚に戻る |
| 3 | 本棚カード | **表紙＋書名＋見開き数＋ミニ目次チップ**（tagRanges流用）。チップクリック→章頭見開きへ |
| 4 | ビューア強化 | **スライダー＋ページ番号クリックで番号入力ジャンプ** を追加 |
| 5 | 見た目 | **「紙の書斎」**：既存ライトテーマのまま、木質×琥珀の棚板。書籍ブランド色を背表紙ライン・チップに反映 |
| 6 | スコープ | **書籍検索.html 1ファイルのみ**。宗教ライブラリ.html・009側は変更しない |

## 3. 対象ファイルと現状構造

```
014_宗教/
├── 書籍検索.html          ← 唯一の変更対象
├── モックアップ_本棚.html  ← 画面イメージ＋移植元CSS/JS（実装完了後に削除）
└── books/                 ← gitignore対象・変更しない
    ├── data/books_index.js   BOOKS_INDEX.tagRanges = {宗教名: [{book,from_page,to_page,chapter}]}
    └── data/book_*.js        BOOK_XXX.meta = {key,title,color,pdfPath,totalSpreads}, pages[]
```

既存コードの押さえどころ（書籍検索.html・行番号は2026-07-04時点）:

- `BOOK_DEFS`（L232付近）: key / short / cls(bk1〜bk4) / ref()。**colorは持たない**→書籍色は `def.ref().meta.color` から取る
- `BOOK_PAGES[key]` = pages配列。**pages[i].page は 1始まり連番＝index+1**（4冊すべて実測検証済み）。
  検索カードは `openViewerFromPage(bookKey, p.page - 1)` → 章頭ジャンプも `from_page - 1` でよい
- pages[0] に `thumb`（`books/img/{key}/thumb/p001.webp`・480px）と `img`（1600px）がある → **表紙 = BOOK_PAGES[key][0].thumb**
- `renderResults(results, rawQ)`（L397付近）: `if (!rawQ) { el.innerHTML=''; ... }` ← **本棚の差し込み点**
- URLパラメータ（L275付近）: `?rel=` 単独着地は現状空白表示 → 本棚導入で自然に解消
- ビューア: `renderViewer()` / `viewerNav(dir)` / keydownリスナー（Escape/←/→）
- ⚠️ **既存 `.v-nav` 内に `<span class="pg" id="vPg">`（L210付近）が既にある**（後述Step 4で置換）
- `renderViewer()` は検索結果・テーマ読み比べ（L602付近）の両方から呼ばれる共通経路。
  **スライダーの max/色の更新は renderViewer 側に置く**（openViewerFromPage側に置くとテーマ経由で更新漏れする）

## 4. 実装ステップ

### Step 1: CSS追加
モックアップ_本棚.html の「★ここから新規：本棚エントランス」ブロックと
「★ビューア」内のスライダー関連（`.v-slider-row` / `.v-pg-jump` / `.v-pg-input`）をコピーして追加。
`:root` に追加: `--shelf-wood:#8A6A44; --shelf-wood2:#6E5233; --shelf-amber:#C9A84C;`

構造上のポイント（レビュー反映済み）:
- 本棚は**単一グリッド** `.shelf-grid` ＋ `.shelf-unit{display:contents}` 方式。
  表紙=row1／棚板=row2（全列ぶち抜き）／書誌情報=row3。**≤820pxでは unit をflex縦カード化し、棚板は本ごとのセグメント（::after）に切替**。2×2時に表紙と書名の対応が崩れない
- `.toc-chip:hover` の背景は `color-mix(... 78%, black)` で白文字コントラストを確保
- `.v-slider-row` が `border-top` を持つ（既存 `.v-nav` の border-top は削除して区切りを一本化）
- `color-mix()` 使用（Chrome 111+。Chrome運用なのでOK）

### Step 2: `buildShelfData()` / `renderShelf(el)` を新設
モックの `MOCK_BOOKS` は使わず実データから導出:

```js
function buildShelfData() {
  return BOOK_DEFS.map(def => {
    const book = def.ref();
    if (!book) return null;
    // tagRanges → 目次チップ（from_page昇順）
    const raw = [];
    for (const [rel, ranges] of Object.entries(BOOKS_INDEX.tagRanges || {})) {
      const r = ranges.find(x => x.book === def.key);
      if (r) raw.push({ rel, page: r.from_page, chapter: r.chapter });
    }
    raw.sort((a, b) => a.page - b.page);
    // ★同一ページ集約：同じ from_page を指すタグが3つ以上なら1チップ「そのほかの宗教」に統合
    //   （実データ: nemure は7タグが p51「第5章 世界のそのほかの宗教」を指す）
    //   2つなら「タグA・タグB」で連結。title属性には chapter と元タグ一覧を入れる
    const toc = mergeSamePage(raw);
    return {
      key: def.key, cls: def.cls,
      title: book.meta.title, color: book.meta.color,
      spreads: book.meta.totalSpreads,
      cover: (BOOK_PAGES[def.key] && BOOK_PAGES[def.key][0]) ? BOOK_PAGES[def.key][0].thumb : '',
      toc,
    };
  }).filter(Boolean);
}
```

描画ルール:
- 書名・chapter等の動的文字列は **必ず `escH()` を通す**
- 表紙 `<button class="cover-btn">`＋`<img onerror=プレースホルダー>`（モック参照。キーボードフォーカス可能に）
- 表紙・書名クリック → `openViewerFromPage(key, 0)`／目次チップ → `openViewerFromPage(key, page - 1)`
- チップに `data-rel="${rel}"` を付与（Step 3のハイライトで使う。統合チップは元タグ全部をスペース区切り等で保持）
- 見出し「📚 N冊・M見開きから読み始める」は `BOOK_DEFS.length`（ref()が生きてる数）と `TOTAL_SPREADS` から動的生成

### Step 3: `renderResults` の空クエリ分岐を本棚に差し替え

```js
if (!rawQ) {
  renderShelf(el);
  countEl.textContent = `全 ${TOTAL_SPREADS} 見開き`;
  updateTagCounts([]);
  if (activeRelTag !== 'all') highlightShelfChips(activeRelTag);
  return;
}
```

- `highlightShelfChips(rel)`: `data-rel` に一致する `.toc-chip` に `.hl` クラス。
  「?rel=仏教で着地→どの本のどこに仏教章があるか一目で分かる」となり、旧・空白問題も改善
- 書籍チップ（activeBooks）で除外中の本は `.shelf-unit.dimmed`（opacity .35）で薄表示（消さない）
- 本棚のフェードインは**初回のみ**（`.anim` クラスを初回だけ付与。検索⇔本棚の往復では再生しない）

### Step 4: ビューアにスライダー＋番号ジャンプ
モックの `.v-slider-row` ブロックを `.v-img-wrap` と `.v-nav` の間に挿入。

- ⚠️ **既存 `.v-nav` 内の `<span class="pg" id="vPg">` は削除**（ID重複防止。ページ表示はスライダー行の `.v-pg-jump` に一本化）
- 変数名の対応（モック→実装）: `vBook.spreads` → `BOOK_PAGES[vBookKey].length`／`vIdx` → `vPageIdx`／
  書籍色 → `def.ref().meta.color`（pdfUrl取得のIIFE（L518付近）と同様の取り方でよい）
- `renderViewer()` に追記: `slider.max = pages.length - 1`／`slider.value = vPageIdx`／accent色の更新
  （openViewerFromPage側でなくrenderViewer側に置く理由: テーマ読み比べ経由でも通る共通経路のため）
- **スライダーは二段イベント**: `input` = 番号表示のみ更新（軽量・画像を読まない）／`change` = renderViewer()。
  ドラッグ中に1600px WebPを連続ロードしない
- 番号ジャンプ: `#vPg` クリック→number入力、Enterで `vPageIdx = 入力-1; renderViewer()`、Esc/blurで表示に戻す
- **keydownガード（確定版）**:
  ```js
  if (e.target.tagName === 'INPUT') {
    if (e.key === 'Escape' && e.target.type !== 'number') closeViewer(); // スライダー上のEscは閉じる
    return;  // ←→の二重送り・番号入力との競合を防ぐ
  }
  ```
  （number入力のEscは入力側のonkeydownが処理して表示に戻す）

### Step 5: 動作確認 → 後片付け → git push
受け入れ条件（下記）を全て確認後、モックアップ_本棚.html を削除してから push。

## 5. 受け入れ条件（チェックリスト）

- [ ] 開いた直後（検索語空）に本棚が表示される。4冊の実物表紙＋書名＋見開き数＋目次チップ
- [ ] 表紙クリック → ビューアが p1 で開き、←→で通読できる
- [ ] 目次チップクリック → その章の先頭見開きが開く（chapter名がツールチップに出る）
- [ ] nemure の目次チップに同一ページの重複がない（p51系は「そのほかの宗教」1チップに統合）
- [ ] 検索語入力 → 本棚が消え検索結果（既存挙動）。×クリア → 本棚に戻る（アニメ再生なし）
- [ ] スライダーで任意ページへ移動できる。ドラッグ中は番号のみ更新、離すと画像ロード
- [ ] 「n / N」クリック→番号入力→Enterでジャンプ。入力中の←→でページ送りされない
- [ ] スライダー操作直後に Esc でビューアが閉じる／←→で二重送りしない
- [ ] `?rel=仏教` で着地 → 本棚表示＋各書籍の仏教チップが強調表示
- [ ] `?q=ジハード` で着地 → 従来どおり検索結果（本棚は出ない）
- [ ] `?rel=仏教&q=ジハード` 同時指定 → 本棚は出ず、タグフィルタ付き検索結果
- [ ] 書籍チップを一部解除して検索クリア → 本棚で該当書籍が薄表示（エラーなし）
- [ ] 書籍チップ全解除＋検索語空 → 本棚が全冊薄表示でエラーなし
- [ ] ウィンドウ幅を820px以下に縮小 → 2×2の縦カード配置。表紙の真下にその本の書名・チップがある
- [ ] テーマ読み比べモードのセルクリック→ビューア → スライダーのmax・色が正しい
- [ ] 宗教ライブラリ.html・009連携（?q=リンク）に影響なし
- [ ] 表紙画像のみ欠落した場合（imgフォルダ不完全）→ プレースホルダー表紙で崩れない
  ※ books/ 全体が無い別PC環境では既存の「books_index.jsが見つかりません」表示になる（既存挙動・対象外）

## 6. 注意事項

- **file:// 直開きで動くこと**。fetch/XHRは使わない（既存の `<script src>` 読み込み方式のまま）
- books/ 配下は gitignore 対象。コミットに含まれないことを確認してから push
- フォントは既存指定（UD デジタル 教科書体 NK 系）を変更しない
- 検索窓の仕様（max-width 320px・×ボタン）は既存準拠、変更しない
- ビューアのバッジ色は既存の `def.cls`（bk1〜bk4クラス）方式を維持（モックのインラインstyleは使わない）
- ナビボタン文言は既存の「← 前の見開き／次の見開き →」を維持
- 既存の検索・テーマ読み比べのコードパスは触らない
  （例外: renderResultsの空クエリ分岐／keydownガード／.v-nav内の旧vPg削除の3点のみ）
